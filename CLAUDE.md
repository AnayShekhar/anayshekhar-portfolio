# ANAY — COMPUTATIONAL NOTEBOOK

## WHAT THIS WEBSITE IS

This website is not a portfolio.

It is not a product.

It is not a startup landing page.

It is not an interactive experiment.

It is a computational notebook.

The website should feel like opening a research notebook that belongs to someone who spends their time understanding how systems work.

Every design decision should support that feeling.

The objective is not to impress visitors.

The objective is to communicate how I think.

When visitors leave, they should understand:

* what I care about
* how I approach problems
* what I am currently exploring

The website should feel calm, intelligent, deliberate, and deeply personal.

# HERO OBJECT

Replace the current blob concept entirely.

The hero should contain a single computational material.

This object is not:

* a sphere
* a blob
* a crystal
* a planet
* a sculpture

The object should feel like:

* suspended density
* liquid intelligence
* ink trapped in water
* information slowly organizing itself

The object should never become perfectly circular.

The object should never become perfectly symmetrical.

The object should never look like a generic 3D asset.

---

# OBJECT BEHAVIOR

The object behaves like a physical material.

Never connect scroll directly to:

* scale
* squash
* stretch
* rotation

Scrolling should not visibly resize the object.

Scrolling should not visibly deform the object.

Scrolling should not feel mechanical.

Instead:

scroll velocity creates energy

energy creates turbulence

turbulence slowly settles

When scrolling stops:

the object continues settling.

Target settling time:

1-2 seconds.

The object should feel heavy.

The object should feel alive.

The object should never feel attached to the page.
